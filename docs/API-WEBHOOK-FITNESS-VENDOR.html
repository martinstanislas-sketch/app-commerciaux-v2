1. Informations de connexion

URL de base
https://VOTRE_DOMAINE
Authentification
Bearer Token
Format
JSON (application/json)

Header obligatoire (sur chaque requête)

Authorization: Bearer wh_mycoachginkgo_2026_secret_key
Content-Type: application/json
Important : Chaque appel doit inclure le header Authorization avec la clé API. Sans ce header, la requête sera rejetée avec une erreur 401.

2. Créer une vente

POST /api/webhook/sales

Envoie une vente unique vers le suivi commerciaux. A appeler chaque fois qu'un commercial saisit une vente dans Fitness Vendor.

Corps de la requête (JSON)

{
  "date": "2026-03-03",
  "amount": 1500,
  "client_first_name": "Jean",
  "client_last_name": "Dupont",
  "commercial_name": "Marvin",
  "rib_status": "En attente"
}

Détail des champs

Champ Type Requis Description
date string Oui Date de la vente au format YYYY-MM-DD
amount number Oui Montant de la vente en euros (nombre positif)
client_first_name string Non Prénom du client
client_last_name string Non Nom du client
commercial_name string Oui* Nom du commercial (insensible à la casse)
external_id string Oui* Identifiant externe du commercial
sales_rep_id number Oui* ID interne du commercial (1, 2 ou 3)
rib_status string Non "Reçu", "En attente" ou "Non fourni" (défaut)
* Identification du commercial : Au moins un de ces trois champs est requis : commercial_name, external_id ou sales_rep_id. Nous recommandons commercial_name pour la simplicité.

Réponse succès

201 Created

{
  "success": true,
  "id": 42,
  "week_start": "2026-03-02",
  "sales_rep_id": 1,
  "rib_status": "En attente"
}

3. Créer plusieurs ventes (bulk)

POST /api/webhook/sales/bulk

Envoie un lot de ventes en une seule requête (maximum 100).

Corps de la requête

{
  "sales": [
    {
      "date": "2026-03-03",
      "amount": 1500,
      "client_first_name": "Jean",
      "client_last_name": "Dupont",
      "commercial_name": "Marvin",
      "rib_status": "Reçu"
    },
    {
      "date": "2026-03-03",
      "amount": 800,
      "client_first_name": "Marie",
      "client_last_name": "Martin",
      "commercial_name": "Magali"
    }
  ]
}

Réponse succès

201 Created

{
  "total": 2,
  "success": 2,
  "failed": 0,
  "results": [
    { "index": 0, "success": true, "id": 43, "week_start": "2026-03-02" },
    { "index": 1, "success": true, "id": 44, "week_start": "2026-03-02" }
  ]
}
Note : Si certaines ventes échouent (champ manquant, commercial non trouvé...), les autres sont quand même insérées. Le champ results détaille le statut individuel de chaque vente.

5. Assigner un identifiant externe

PUT /api/webhook/sales-reps/:id

Associe un identifiant Fitness Vendor à un commercial. Utile si les noms diffèrent entre les deux systèmes.

Corps de la requête

{
  "external_id": "FV-001-MARVIN"
}

Réponse succès

200 OK

{
  "success": true,
  "id": 1,
  "external_id": "FV-001-MARVIN"
}

6. Mapping des champs

Correspondance entre les champs de Fitness Vendor et ceux de notre API :

Champ Fitness Vendor Champ Webhook Notes
Prénom du client → client_first_name Texte libre
Nom du client → client_last_name Texte libre
Montant de la vente → amount Nombre en euros (ex : 1500)
Date de la vente → date Format YYYY-MM-DD obligatoire
Nom du commercial → commercial_name Marvin, Magali ou Fabian
Statut du RIB → rib_status Reçu, En attente ou Non fourni

7. Valeurs Statut RIB

Valeur Affichage Description
"Non fourni" Non fourni RIB non encore demandé (valeur par défaut si le champ est omis)
"En attente" En attente RIB demandé, pas encore reçu
"Reçu" Reçu RIB reçu et validé

8. Configuration dans Bubble.io (Fitness Vendor)

1

Installer le plugin API Connector

Dans l'éditeur Bubble → Plugins → Add plugins → Rechercher et installer API Connector

2

Configurer l'API

Dans le plugin API Connector :

  • Cliquer Add another API
  • Nom : MyCoachGinkgo
  • Authentication : Private key in header
  • Key name : Authorization
  • Key value : Bearer wh_mycoachginkgo_2026_secret_key
3

Ajouter l'appel "Créer une vente"

Cliquer Add another call avec ces paramètres :

  • Nom : Envoyer Vente
  • Méthode : POST
  • URL : https://VOTRE_DOMAINE/api/webhook/sales
  • Header : Content-Type: application/json
  • Body type : JSON
{
  "date": "<date_de_la_vente>",
  "amount": <montant>,
  "client_first_name": "<prenom_client>",
  "client_last_name": "<nom_client>",
  "commercial_name": "<nom_du_commercial>",
  "rib_status": "<statut_rib>"
}

Remplacer les valeurs entre < > par les champs dynamiques Bubble correspondants.

4

Déclencher l'appel automatiquement

Dans le Workflow Bubble :

  • Event : When a new sale/vente is created
  • Action : Plugins → MyCoachGinkgo - Envoyer Vente
  • Mapper les champs dynamiques de la vente vers les paramètres de l'appel
Résultat : Dès qu'un commercial saisit une vente dans Fitness Vendor, elle apparaît automatiquement dans le tableau de bord Suivi Commerciaux avec tous les KPIs mis à jour en temps réel.

9. Codes de réponse

Code Signification Quand
200 OK Requête GET ou PUT réussie
201 Created Vente(s) créée(s) avec succès
400 Bad Request Champs manquants, format de date invalide, montant négatif
401 Unauthorized Clé API manquante ou invalide
403 Forbidden Semaine verrouillée (saisie fermée pour cette semaine)
404 Not Found Commercial non trouvé
409 Conflict L'external_id est déjà attribué à un autre commercial

10. Exemples de test cURL

Créer une vente

curl -X POST https://VOTRE_DOMAINE/api/webhook/sales \
  -H "Authorization: Bearer wh_mycoachginkgo_2026_secret_key" \
  -H "Content-Type: application/json" \
  -d '{
    "date": "2026-03-03",
    "amount": 1500,
    "client_first_name": "Jean",
    "client_last_name": "Dupont",
    "commercial_name": "Marvin",
    "rib_status": "En attente"
  }'

Tester l'authentification (doit retourner 401)

curl -X POST https://VOTRE_DOMAINE/api/webhook/sales \
  -H "Content-Type: application/json" \
  -d '{"date":"2026-03-03","amount":100,"commercial_name":"Marvin"}'

Lister les commerciaux

curl -X GET https://VOTRE_DOMAINE/api/webhook/sales-reps \
  -H "Authorization: Bearer wh_mycoachginkgo_2026_secret_key"

Envoyer plusieurs ventes

curl -X POST https://VOTRE_DOMAINE/api/webhook/sales/bulk \
  -H "Authorization: Bearer wh_mycoachginkgo_2026_secret_key" \
  -H "Content-Type: application/json" \
  -d '{
    "sales": [
      {"date":"2026-03-03","amount":1500,"commercial_name":"Marvin","client_first_name":"Jean","client_last_name":"Dupont"},
      {"date":"2026-03-03","amount":800,"commercial_name":"Magali","client_first_name":"Marie","client_last_name":"Martin"}
    ]
  }'
